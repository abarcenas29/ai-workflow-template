## Architectural Pattern

**Agent-based workflow distribution system.** The project is a template package that distributes AI-assisted development configuration files. It follows a layered architecture:

1. **Instructions Layer** (`.agents/instructions/`) — Coding standards, patterns, and frameworks
2. **Skills Layer** (`.agents/skills/`) — Reusable task-based workflows
3. **Agents Layer** (`.opencode/agents/`) — Specialized AI agents with defined roles
4. **Prompts Layer** (`.agents/prompts/`) — Reusable prompt templates
5. **Scripts Layer** (`scripts/`) — Automation and sync utilities
6. **Memory Bank** (`memory-bank/`) — Persistent project context across sessions

## Dependency Rules

- Instructions are stateless and read-only references
- Skills may depend on instructions but not on other skills
- Agents may depend on skills and instructions
- Orchestrators delegate to agents via subagent_type
- Memory bank files build hierarchically: projectbrief → productContext/systemPatterns/techContext → activeContext → progress/tasks

## Notable Deviations / Gotchas

- This is a template distribution package (not a deployable application)
- Source code is configuration/skill/agent files, not traditional application code
- The `scripts/` directory contains the only executable JavaScript — this is where unit tests apply
- E2E tests in `tests/` validate template structure correctness
- The TDD orchestrator requires a unit test framework (Vitest) — added during infrastructure bootstrap

## Extension Points

- Add new agents in `.opencode/agents/` following existing naming conventions
- Add new skills in `.agents/skills/<name>/SKILL.md`
- Add new instructions in `.agents/instructions/<name>.instructions.md`
- Extend scripts in `scripts/` following existing ES module patterns
- Run `graphify update .` after any code change to keep the knowledge graph current
